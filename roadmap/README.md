# roadmap/

The public-facing counterpart to mvp/ — what's coming, in plain
language, without internal implementation detail. This is what gets
adapted for public/ build-in-public posts; mvp/ stays the internal
checklist.

## What belongs here

- Phase-by-phase roadmap in outcome language ("reps get a single
  prioritized queue" not "implement fetch_all_leads with pagination")
- Rough timeline, if you're comfortable sharing one
- What's explicitly not planned yet (manages expectations)

## Suggested file naming

  roadmap.md              Current public roadmap

## Draft Phase 1 roadmap (derived from PRD, needs a public-friendly pass)

**Phase 1 — Core triage and handoff**
One consolidated view of every lead across all your intake sheets,
automatically prioritized so nothing falls through the cracks.
Automatic tracking of every call, text, and email touch so reps never
duplicate outreach. Instant Slack alerts to the back office the moment
a deal file is complete.

**Later (not yet scoped)**
Multi-org support, richer analytics, CRM integrations beyond Google
Sheets.

## Notes

- Keep this in outcome/benefit language — no tool names, no API
  names, no internal architecture. Compare against prd/ and mvp/ if
  unsure what's safe to say publicly.
- Update alongside public/ posts, not as a substitute for them.
